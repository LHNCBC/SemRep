14.18.2 Bugs Fixed

  • Terms containing a hash mark (‘#’) was not transmitted correctly to the SICStus Prolog IDE (SPIDER). This affected the SPIDER debugger view, among other things.
  • Several built-in predicates did not handle constrained “output” variables correctly, e.g. atom_length/2, char_code/2, atom_concat/3, sub_atom/5
  • Fix variable binding display bug when inside disjunctions and do-loops
  • Prevent state corruption upon out-of-memory exception in certain contexts
  • library(random): random(L,L,X) would succeed with X = L instead of failing
  • queues:singleton_queue/2 returned the wrong term
  • CLPFD:
    • - Performance bug in special cases of case/[3,4]
    • - table/[2,3]: bug with inf/sup in binary relations
